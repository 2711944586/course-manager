# 一键启动脚本改进说明

## 文件位置
`start-course-manager.bat`

## 改进内容

### 1. **后端端口可用性检测**
- ✅ 启动后端后，脚本会主动检测 8000 端口是否可用（最多等待 20 秒）
- ✅ 确保后端真正启动成功后，再启动前端
- ❌ 如果后端启动失败，会给出清晰的故障提示（仍会继续启动前端）

**技术实现**: 使用 PowerShell 的 `TcpClient` 每秒测试一次端口连接

### 2. **清晰的启动阶段划分**
脚本现在分为三个阶段，每个阶段都有明确的标识：
- `[初始化]` — Python/npm/依赖检查
- `[启动中]` — 后端/前端启动
- `[关闭中]` — 服务停止

### 3. **视觉化提示优化**
- ✓ 成功标识 — `✓`
- ⚠ 警告标识 — `⚠` 
- ✗ 失败标识 — `✗`
- 统一的美观头部横幅

### 4. **更好的错误处理**
- Python 环境缺失 → 明确提示路径，继续启动前端
- npm 缺失 → 给出下载链接和超时提示
- 依赖安装失败 → 清晰的错误提示
- 后端启动失败 → 提示检查端口占用情况

### 5. **服务关闭优化**
- 使用 `taskkill` 精准关闭 `Aurora-Backend` 窗口
- 避免进程残留

## 使用方法

### 正常启动
1. **双击运行** `start-course-manager.bat`
2. 脚本自动检查所有环境
3. 自动启动后端（如有 Python）和前端
4. 浏览器自动打开前端界面（http://127.0.0.1:4200）
5. **关闭脚本窗口** 即停止所有服务

### 故障排查

#### 后端启动失败，但前端启动成功
- 检查 Python 环境是否完整
- 检查 8000 端口是否被其他应用占用
- 查看后端窗口的错误信息

#### 前端依赖安装失败
- 检查网络连接
- 确保 npm 已正确安装
- 尝试手动运行 `npm install`

#### 无法关闭后端进程
- 手动打开任务管理器，搜索 `Aurora-Backend` 窗口并结束任务
- 或者运行 `taskkill /fi "WINDOWTITLE eq Aurora-Backend*" /f`

## 技术细节

### 后端端口检测原理
```batch
powershell -Command "try { [System.Net.Sockets.TcpClient]::new('127.0.0.1', 8000).Dispose(); exit 0 } catch { exit 1 }"
```
- 尝试创建 TCP 连接到 127.0.0.1:8000
- 连接成功则返回 0，失败则返回 1
- 每秒重试一次，最多 20 次（20 秒超时）

### 为什么要等待后端？
- **避免竞态条件**: 前端可能在后端完全就绪前尝试调用 API
- **更好的用户体验**: 用户不용看到 API 连接错误
- **调试更清晰**: 能够识别真正的后端故障 vs 启动时序问题
